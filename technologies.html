Технологічний стек проекту

Програмний комплекс для аналізу пошти на спам використовує сучасні технології Python для забезпечення точної класифікації повідомлень та зручного інтерфейсу користувача.

Основні технології

Python 3.9+

Основна мова програмування проекту, обрана завдяки широким можливостям для машинного навчання.

  • Простота та читабельність коду
  • Багата екосистема бібліотек
  • Крос-платформеність

PyQt5

Бібліотека для створення графічного інтерфейсу з таблицею листів та панеллю управління.

  • Мультиплатформений інтерфейс
  • Підтримка багатопоточності
  • Сучасний вигляд елементів

Scikit-learn

Основна бібліотека для реалізації алгоритмів машинного навчання.

  • Наївний Байєс (MultinomialNB)
  • Випадковий ліс (Random Forest)
  • Логістична регресія
  • TF-IDF векторизація

Додаткові бібліотеки

IMAPLib

Для підключення до Gmail через IMAP-протокол та отримання листів

Pandas

Обробка датасету для навчання моделей (spam_dataset.csv)

Joblib

Серіалізація та збереження навчених моделей

Plyer

Система сповіщень про завершення перевірки пошти

Re (регулярні вирази)

Очищення тексту листів від HTML-тегів та зайвих символів

Архітектура системи

Основні компоненти

  1. Графічний інтерфейс - PyQt5, відображає листи, результати перевірки та дозволяє керувати процесом
  2. IMAP-клієнт - отримує листи з Gmail-акаунта користувача
  3. Модуль обробки листів - витягує текст, тему та відправника
  4. Система класифікації - 4 алгоритми для визначення спаму
  5. Історія перевірок - збереження результатів у текстові файли
  6. Менеджер спаму - переміщення виявленого спаму до відповідної папки

Процес обробки листів

Отримання листа

  • Підключення до Gmail через IMAP
  • Завантаження заголовків (відправник, тема)
  • Витяг текстового вмісту (з HTML або plain text)

Попередня обробка

  • Видалення HTML-тегів
  • Очищення від спецсимволів
  • Перетворення до нижнього регістру
  • Конкатенація теми та тіла листа

Аналіз та класифікація

  • Перевірка на ключові фрази (кастомний Байєс)
  • Класифікація обраним алгоритмом ML
  • Визначення ймовірності спаму
  • Прийняття рішення на основі порогового значення

Навчання моделей

Для тренування моделей використовується датасет spam_dataset.csv з розміткою листів на спам/не спам.

Етап Опис Використані інструменти
Завантаження даних Читання CSV-файлу з текстами листів та мітками Pandas
Векторизація Перетворення текстів у числові вектори TfidfVectorizer
Тренування моделі Навчання обраного алгоритму на підготовлених даних Scikit-learn
Оцінка якості Перевірка точності на тестовій вибірці Classification report
Збереження моделі Серіалізація для подальшого використання Joblib
